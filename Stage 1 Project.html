Important Concepts

Lesson 1: Important Concepts in HTML

How the Web Works

The web consists of a huge number of interconnected computers that communicate with each other in order to deliver content from a source to a user who seeks it. When a user navigates to a given site - say Alta Vista - their web browser sends a HTTP Request to a server, which then returns the appropiate HTML document back to the user. The user's web browser interprets the HTML and displays the page on screen as it is meant to be seen by the user!

This video does a good job of explaining more.

HTML

HTML stands for Hypertext Markup Language. HTML documents form the majority of the content on the web. HTML is made up of text content, markup, references to other documents, and links. The text content describes "what you see" and the markup describes "how it looks."

this video does a good job of explaining.

Tags and Elements

In HTML, we use tags to markup, or style, the content. For example, we can bold our text, or use italics to place an emphasis on something important just by using tags to alert the web browser how to display the text content.

This video explains the distinction well.

Why Computers are Stupid

Computers don't use any judgement when displaying HTML for the user. Any programming mistakes will result in the display of an end product that does not reflecting what was meant by the programmer!

Inline vs Block Elements

HTML elements are either inline or block. Block elements, which include paragraph, form, and div, possess an "invisible box" around the content inside of them. Inline elements do not possess this invisible box. Some examples of inline elements are: span, break, image, bold, and anchor.

Characteristics of block elements will be important for properly styling the pages I want to create laterx.

Developer Tools (in the Browser)

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.

The "tree-like structure" of HTML

The "tree-like structure" comes from the fact that HTML elements can have other elements inside of them. You can draw this relationship like a family tree.

In a browser, this structure shows up as a series of nested boxes. There are boxes inside of boxes inside of boxes, and so on...

Text Editors (for programming)

When writing code, programmers use special text editors. I am using Sublime Text. These editors make the programmer's life easier by automatically generating a closing HTML tag when you write an opening tag.

Lesson 3: Adding CSS Style to HTML Structure

Limit Repetition

Avoiding repetition is important for a variety of reasons.

Avoiding Errors: It's important to take steps to limit repetition within code in order to simplify the work and minimize mistakes. Rewriting or copy/pasting code, whether html or style attributes, both increases the liklihood of errors and makes them more difficult to locate and debug.

Consistency: Instead of manually giving each section of this site the same style, we can assign each lesson div the same class and then specify the style for that class the CSS stylesheet. This helps elimitate repitition and simplifies the process of editing the site's style.

CSS: Cascading Style Sheets

CSS gives programmers a way to control the style of related HTML elements. This is done by giving similar HTML elements the same class name and then specifying the style that should apply to that class.
